# Пояснение к файлам портфолио

Данные три файла составляют портфолио для демонстрации навыков написания пояснительных/технических текстов к IT-проектам. Каждый из них демонстрирует текст разного формата, цели и стилистики написания. Код всех проектов, по которым были написаны данные пояснительные тексты, был также написан мной лично. Таким образом, мой стандартный цикл работы представляет собой:

Получение задачи -> Проектирование архитектуры, логики и интерфейса -> Разработка программного продукта -> Тестирование -> Написание пояснительных текстов к нему

## Документация к вебсайту

**Описание проекта:**

Проектом является вебсайт для аренды книг с двумя уровнями доступа: клиентским и администраторским. Это была работа, сделанная на заказ для студентки. Проект должен обеспечивает экономику аренды книг, включая штрафы за невозврат/возврат испорченных книг, удержание депозита, проведение операций с балансом пользователя, отображение наличия книг, несколько филиалов, несколько стадий доставки и пр. Со стороны администраторской части приложения осуществляется изменение статуса доставки, работа с каталогом книг и пр.

*Исходный код данного проекта может быть [доступен к просмотру на GitHub](https://github.com/juliafox6/websitenov)*

**Язык разработки:**

Стек `HTML`, `CSS`, `JavaScript`; `Flask` для клиент-серверного взаимодействия, локальная БД, реализованная через `SQLite`

**Цель работы:**

Описать полную структуру проекта через разбор репозитория и ключевых файлов, объяснить логику взаимодействия, структуру страниц, шаблонов и функций для клиента, которому передавали проект. Объяснить некоторые концепции разработки на примере написанного кода и принципы проектирования вебсайта, дать инструкцию к запуску, пользованию и редактированию

**Стилистика текста:**

Наиболее приближенный к технической документации, язык разметки Markdown

**Целевая аудитория:**

Разработчики

## Пояснение к модели передвижения автомобилей и пешеходов

**Описание проекта:**

Данное пояснение написано к модели движения автомобилей и пешеходов на нерегулируемом перекрестке. Данный проект включает в себя логику взаимодействия машин (назначение начальных и конечных точек, простроение маршрутов, уступание дороги, задержка перед пешеходным переходом) и пешеходов (аналогичный функционал, однако вместо уступания дороги - остановка и ожидание перед пешеходным переходом), а также несколько событий в модели, такие как поломка машины или появление на проезжей части полицейской машины, которой необходимо уступить дорогу. Также одним из этапов разработки был самостоятельный дизайн графических объектов в формате пиксель-арта. Проект был разработан на заказ для студентки

*Исходный код данного проекта может быть [доступен к просмотру на GitHub](https://github.com/juliafox6/crossroadModel)*

**Язык разработки:**

Чистый `C++/CLI`, `.NET` с использованием `Bitmap` и `Graphics` для отрисовки взаимодействия и перемещения объектов по тикам

**Цель работы:**

В данном случае не было необходимости полностью охватывать структуру проекта и весь его функционал, поскольку большая часть пояснений была дана в виде комментариев в коде. Однако данный файл служит пояснительной запиской к защите проекта, поэтому его внимание акцентировано на разборе реализации ключевых требований ТЗ по пунктам: что требуют, как это реализовано, где это находится в коде и как работает по логике языка разработки или конкретного программного решения. Также даны пояснения к некоторым концепциям проектирования, которые могут быть непонятны клиенту, и заметки о ключевых отличительных чертах и особенностях

**Стилистика текста:**

Полу-неформальный, пояснительный, краткое изложение

**Целевая аудитория:**

Студенты

## Фрагменты ВКР

**Описание проекта:**

В данном случае было разработано многостраничное и относительно тяжёлое мобильное приложение для операционной системы Android, которое являлось личной выпускной квалификационной работой для диплома бакалавра. Приложение TaskTrek является объединенной средой для самоорганизации, включающей в себя менеджер задач, трекер привычек и Pomodoro-таймер. Функционал во многом основен на геймификации, а именно: применении игровых механизмов Тамагочи к трекеру привычек, визуализации в формате игры Тетрис для задач, системе уровней и опыта для пользователя и интерфейсе выращивания растений для Pomodoro-таймера (по аналогии с приложением Forest). Помимо основного функционала, в приложении реализована система рекомендаций по продуктивности для пользователя, основанная на детерминированном анализе его статистики (по временным меткам его активности, отслеживаемым по мере выполнения действий в приложении). Одним из этапом была полная самостоятельная разработка UI/UX оригинального интерфейса в Figma и подбор стиля оформления 

*Исходный код данного проекта может быть [доступен к просмотру на GitHub](https://github.com/juliafox6/taskTrek)*

**Язык разработки:**

Основная часть приложения написана на `Kotlin` (и `Java` в legacy-части) в совместной работе с  `XML`, база данных реализована на `Supabase`

**Цель работы:**

Текст служит пояснительной запиской к выпускной квалификационной работе, поэтому его целью является подробное строго формализованное описание приложения целиком. В представленный документ были включены фрагменты наибольшего интереса: описание архитектуры приложения, серверной части и формирования пользовательских рекомендаций. 

**Стилистика текста:**

Подробный, академический, строгий

**Целевая аудитория:**

Научная комиссия
